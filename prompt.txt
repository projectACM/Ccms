### 4. Database & schema.sql

Generate a **schema.sql** file that creates the database tables below:

**users**

* id INT PRIMARY KEY AUTO_INCREMENT
* name VARCHAR(100)
* email VARCHAR(150) UNIQUE
* password_hash VARCHAR(255)
* phone VARCHAR(20) NULL
* created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

**cards**

* id INT PRIMARY KEY AUTO_INCREMENT
* user_id INT NOT NULL (FK to users.id)
* card_name VARCHAR(100) NOT NULL
* bank_name VARCHAR(100) NOT NULL
* last4 VARCHAR(4) NOT NULL
* limit_amount DECIMAL(12,2) NOT NULL
* current_balance DECIMAL(12,2) NOT NULL DEFAULT 0
* statement_date DATE
* due_date DATE
* created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
* updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

Include foreign key:
`FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE`

### 5. Backend Logic (PHP)

**Authentication**

* Registration validates email uniqueness and hashes passwords using `password_hash()`

* Logout destroys session

**Card CRUD**

* All operations require a logged-in user
* Use PDO + prepared statements
* Always scope queries with `WHERE user_id = :user_id`

**User Profile Page**

* Prefill form with existing user info

* Email must remain unique
* Password change optional (only update if entered)
* Use the database and PDO; update only for the logged-in user
